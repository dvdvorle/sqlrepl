# SqlRepl — LLM Instructions

## Project overview
Oracle SQL REPL in C# (.NET 10). Uses Typin for interactive CLI, Spectre.Console for rich output, SQLite for history.

## Architecture
- **Typin commands**: Each command is a class with `[Command("name")]`. The unnamed default command (`SqlCommand`) catches SQL input.
- **DI**: All services registered in `Program.cs` via Typin's `ConfigureServices`. Singletons: `ConnectionManager`, `ConnectionStore`, `IQueryExecutor`, `CommandHistory`, `SqlBuffer`, `ReplSettings`.
- **IQueryExecutor**: Interface extracted from `QueryExecutor` to enable mocking in tests.

## Key patterns
- **NonBreakingText**: Custom `IRenderable` in `ResultRenderer.cs`. Never wraps text at whitespace. Truncates with ellipsis. Has maxWidth guards (0→empty, 1→"…").
- **SqlBuffer**: Singleton multiline accumulator. Lines without trailing `;` are buffered. Semicolon triggers execution.
- **CommandHistory**: SQLite at `~/.sqlrepl/history.db`. Three tables: `commands` (unique), `executions` (per-run), `commands_fts` (trigram FTS5, contentless). Cannot delete from contentless FTS5.
- **ConnectionStore**: JSON at `~/.sqlrepl/connections.json`. Fuzzy search: substring containment (score 100) → `Fuzz.Ratio` (threshold 70).

## Spectre.Console quirks (v0.54.1-alpha.0.86)
- `Markup` constructor crashes if you pass `null` as the `Style?` parameter — avoid it
- `[nobr]` markup tag does NOT exist
- `TableColumn` doesn't implement `IOverflowable`
- `TestConsole.Profile.Capabilities` may throw — wrap in try/catch

## Testing
- 125+ tests using xUnit + NSubstitute + Spectre.Console.Testing
- `VirtualConsole.CreateBuffered()` for command output capture
- `TestConsole` for renderer tests (no ANSI support)
- SQLite tests need `SqliteConnection.ClearAllPools()` in Dispose for cleanup
- Tests use temp directories with unique GUIDs for isolation

## Conventions
- Write failing test first, then implement
- Commands go in `SqlRepl/Commands/`
- Tests go in `SqlRepl.Tests/` named `{Feature}Tests.cs`
- Spectre markup text with user data must use `Markup.Escape()`
- SQL in Oracle metadata queries should use uppercase table names
